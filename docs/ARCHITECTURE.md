# Архитектура проекта

## Краткий обзор
Проект — статический одностраничный сайт‑портфолио на Vite с Bulma и Sass. Архитектура минималистична: один HTML‑каркас, один JS‑вход, единая точка стилей. Серверной логики нет.

## Карта системы (уровни)
1. Runtime: Node.js — запускает инструменты сборки.
2. Package manager: npm/yarn — управляет зависимостями.
3. Bundler: Vite — dev‑сервер и production‑сборка.
4. Styles: Sass — переменные и структура стилей.
5. UI: Bulma — сетка и компоненты.

## Диаграмма потоков (упрощенно)
```
index.html
   │
   ├── подключает src/main.js
   │        ├── импортирует src/styles/main.scss
   │        ├── управляет меню и reveal‑эффектами
   │        └── обновляет [data-year]
   │
   └── содержит секции и разметку
```

## Структура каталогов
```
project/
  index.html
  package.json
  vite.config.js
  public/
  src/
    main.js
    assets/
    styles/
      _variables.scss
      _base.scss
      _layout.scss
      _components.scss
      _utilities.scss
      main.scss
  docs/
  practice/
```

## Входные точки
- HTML: `index.html` — основной каркас страницы и секций.
- JS: `src/main.js` — подключение стилей и минимальная логика UI.
- Styles: `src/styles/main.scss` — точка сборки Sass.

## Потоки данных
- Данных как таковых нет: сайт статический.
- JS только читает DOM и добавляет классы для интерфейса.

## Поведение на клиенте
- Подстановка года в футере (`[data-year]`).
- Переключение мобильного меню (`.navbar-burger` + `#mainNav`).
- Reveal‑анимации через `IntersectionObserver`.

## Сборка и запуск
- `npm run dev` — dev‑сервер.
- `npm run build` — production‑сборка.
- `npm run preview` — локальный preview.

## Конвенции
- Sass‑переменные определяются до импорта Bulma.
- Структура Sass разделена по ответственности.
- Кастомные UI‑блоки добавляются в `_components.scss`.

## Границы ответственности
- `index.html` — структура и семантика.
- `src/styles/` — визуальная система и компоненты.
- `src/main.js` — только поведение интерфейса (без бизнес‑логики).

## Риски и ограничения
- Нет SSR/SEO‑оптимизаций, кроме статической HTML‑страницы.
- JS логика минимальна, но критична для меню и анимаций.
 - Возможны предупреждения Dart Sass о депрекации `@import` и устаревших функций Bulma 0.9.4. Это не влияет на работу, но в будущем потребуется миграция на Sass‑модули и/или обновление Bulma.

## Точки расширения
- Добавление секций: `index.html` + `_components.scss`.
- Новые визуальные темы: `_variables.scss`.
- Новые эффекты появления: `.reveal` + CSS.
