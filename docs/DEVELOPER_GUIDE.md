# Руководство разработчика

## Быстрый старт

```bash
npm install
npm run dev
```

## Скрипты
- `npm run dev` — dev‑сервер.
- `npm run build` — production‑сборка.
- `npm run preview` — предпросмотр сборки.

## Где что менять
- Тексты и секции: `index.html`.
- Логика и эффекты: `src/main.js`.
- Тема и переменные: `src/styles/_variables.scss`.
- Базовые стили: `src/styles/_base.scss`.
- Макеты и сетки: `src/styles/_layout.scss`.
- Компоненты: `src/styles/_components.scss`.
- Утилиты: `src/styles/_utilities.scss`.

## Как добавить новую секцию
1. Добавьте блок разметки в `index.html`.
2. Используйте классы Bulma для сетки и отступов.
3. Добавьте стили в `_components.scss` или `_layout.scss`.
4. Если нужен эффект появления — добавьте класс `.reveal`.

## Стиль и структура Sass
- Переменные Bulma задаются **до** импорта Bulma.
- `main.scss` — точка сборки, порядок импортов важен.
- Новые токены лучше добавлять в `_variables.scss`.

## JS‑логика
В `src/main.js` реализовано:
- автоматическая подстановка года (`[data-year]`),
- переключение burger‑меню (`.navbar-burger` + `#mainNav`),
- reveal‑анимации через `IntersectionObserver`,
- снятие класса `is-preload` после загрузки.

## Процесс разработки (рекомендуемый)
1. Запустить dev‑сервер.
2. Вносить изменения в HTML и Sass.
3. Проверить адаптивность в DevTools.
4. Протестировать сборку (`npm run build`).

## Как работать с дизайном
- Базовые токены меняются в `_variables.scss`.
- Для новых блоков используйте `_components.scss`.
- Для макета секций — `_layout.scss`.

## Troubleshooting (кратко)
- Если не собирается Sass: проверьте порядок импортов в `src/styles/main.scss`.
- Если не работает burger‑меню: проверьте наличие `#mainNav` и `.navbar-burger`.
- Если reveal‑анимации не видны: убедитесь, что элементы имеют класс `.reveal`.
 - Если видите предупреждения Sass про `@import` и устаревшие функции: это депрекации Dart Sass и Bulma 0.9.4. На работу проекта это не влияет, но в будущем потребуется миграция на Sass‑модули и/или обновление Bulma.

Подробные инструкции по инструментам см. в `practice/guides/`.
