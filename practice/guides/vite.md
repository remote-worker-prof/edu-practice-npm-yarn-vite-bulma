# Vite — гайд

## Что это
Vite — современный сборщик и dev‑сервер. Его ключевая идея — быстрый запуск разработки и быстрая сборка.

## Роль в этом проекте
- Запускает dev‑сервер.
- Собирает production‑версию.
- Отвечает за hot‑reload.

## Мини‑теория
Vite использует `index.html` как точку входа. Он строит модульный граф зависимостей и обновляет только те модули, которые изменились. Это делает разработку очень быстрой.

## Основные команды
```bash
npm run dev
npm run build
npm run preview
```

## Что происходит при `dev`
- Vite поднимает сервер на `http://localhost:5173`.
- Читает `index.html`.
- Импортирует `src/main.js` и связанные модули.
- Компилирует Sass через `sass`.
- Отдает результат в браузер и обновляет изменения без перезапуска.

## Что происходит при `build`
- Создается папка `dist/`.
- CSS и JS оптимизируются.
- Итог готов для деплоя.

## Типовые ошибки
- Порт занят.
- Зависимости не установлены.
- Открытие `index.html` напрямую вместо dev‑сервера.

## Как понять, что всё ок
- Dev‑сервер запускается и открывает сайт.
- В `dist/` появляется сборка после `build`.

## Короткий вывод
Vite — сердце проекта. Он объединяет HTML, JS и Sass в рабочий продукт.
