# Yarn — гайд

## Что это
Yarn — альтернативный менеджер пакетов для Node.js. Работает аналогично npm, но использует собственный lock‑файл и чуть другой интерфейс.

## Роль в этом проекте
- Установка зависимостей.
- Запуск Vite‑скриптов.

## Как установить (если yarn нет)
Рекомендуемый способ — через Corepack (идет вместе с Node.js):
```bash
corepack enable
corepack prepare yarn@stable --activate
```

Проверка установки:
```bash
yarn -v
```

Если Corepack недоступен, можно использовать npm:
```bash
npm i -g yarn
```

## Мини‑теория
Yarn читает `package.json`, устанавливает зависимости и запускает скрипты. Он фиксирует версии в `yarn.lock`. Важно выбрать **один** менеджер в проекте.

## Основные команды
```bash
yarn
yarn dev
yarn build
yarn preview
```

## Файл фиксации версий
- `yarn.lock` — фиксирует точные версии зависимостей.

## Типовой рабочий цикл
1. `yarn` — установка зависимостей.
2. `yarn dev` — запуск dev‑сервера.
3. `yarn build` — production‑сборка.
4. `yarn preview` — просмотр собранной версии.

## Типовые ошибки
- Одновременное использование npm и yarn.
- Отсутствие `yarn.lock` после установки.
- Запуск `yarn dev` без установленного Node.js.

## Как понять, что всё ок
- Появился `yarn.lock`.
- `yarn dev` запускается без ошибок.

## Короткий вывод
Yarn — удобная альтернатива npm. Важно соблюдать дисциплину: выбрать один менеджер пакетов и использовать только его.
