# Webpack — гайд для студентов (альтернатива)

## Что это
Webpack — классический сборщик, требующий явной настройки. В учебном проекте рассматривается для сравнения подходов с Vite.

## Когда Webpack уместен
- Большие легаси‑проекты.
- Сложные пайплайны с кастомными лоадерами.
- Монорепозитории со старой инфраструктурой.

## Минимальная идея настройки
- Нужны лоадеры для Sass: `sass-loader` → `css-loader` → `style-loader`.
- Точка входа: `src/index.js`.
- Dev‑сервер: `webpack-dev-server`.

## Сравнение с Vite
- Vite быстрее стартует и проще в конфигурации.
- Webpack гибче в сложных сборках.

## Типовые ошибки
- Использование `node-sass` вместо `sass`.
- Неверный порядок лоадеров.
- Отсутствие конфигурации для статических файлов.

## Мини‑задание
1. Объясните, почему в учебной практике выбран Vite.
2. Перечислите, какие лоадеры нужны для Sass в Webpack.

## Мини‑чеклист
- Понимание отличий Vite и Webpack.
- Умение назвать минимальный набор лоадеров.
