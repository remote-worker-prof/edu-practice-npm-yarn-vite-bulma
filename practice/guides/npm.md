# npm — гайд

## Что это
npm — стандартный менеджер пакетов для Node.js. Он устанавливает зависимости и запускает скрипты из `package.json`.

## Роль в этом проекте
- Установка зависимостей проекта.
- Запуск команд `dev`, `build`, `preview`.

## Мини‑теория
`package.json` — это манифест проекта. В нем есть зависимости и скрипты. npm читает этот файл и выполняет команды. Например, `npm run dev` запускает скрипт `dev`, который в нашем проекте равен `vite`.

## Основные команды
```bash
npm install
npm run dev
npm run build
npm run preview
```

## Файл фиксации версий
- `package-lock.json` — фиксирует точные версии зависимостей.

## Пример рабочего цикла
1. `npm install` — установка зависимостей.
2. `npm run dev` — запуск dev‑сервера.
3. `npm run build` — production‑сборка.
4. `npm run preview` — просмотр собранной версии.

## Типовые ошибки
- Смешивание npm и yarn в одном проекте.
- Удаление `package-lock.json` без причины.
- Запуск `npm run build` до `npm install`.

## Как понять, что всё ок
- Появился `package-lock.json`.
- Создалась папка `node_modules/`.
- Dev‑сервер запускается без ошибок.

## Короткий вывод
npm — базовый и самый простой вариант менеджера пакетов для проекта. Главное правило — использовать только его, не смешивая с yarn.
