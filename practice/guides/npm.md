# npm — гайд для студентов

## Что это
npm — стандартный менеджер пакетов для Node.js. Управляет зависимостями и запускает скрипты из `package.json`.

## Зачем в этом проекте
- Установка зависимостей.
- Запуск `dev`, `build`, `preview`.

## Когда выбирать npm
- Если npm уже установлен вместе с Node.js.
- Если в команде принято использовать npm.

## Минимальные команды
```bash
npm install
npm run dev
npm run build
npm run preview
```

## Где фиксируются версии
- `package-lock.json` — точные версии зависимостей.

## Проверка результата
- Появилась папка `node_modules/`.
- Создан `package-lock.json`.
- `npm run dev` запускается без ошибок.

## Типовые ошибки
- Смешивание npm и yarn в одном репозитории.
- Удаление `package-lock.json` без причины.
- Запуск `npm run build` без `npm install`.

## Мини‑задание
1. Установите зависимости.
2. Запустите dev‑сервер.
3. Сделайте скриншот страницы.

## Мини‑чеклист
- Используется только npm.
- Есть `package-lock.json`.
- Скрипты запускаются без ошибок.
