# Node.js — расширенное руководство

## Что такое Node.js
Node.js — это среда выполнения JavaScript вне браузера. Она построена вокруг событийной модели и неблокирующего ввода‑вывода: почти никакие операции не блокируют поток исполнения, поэтому приложение может обслуживать множество операций параллельно, не «замораживая» процесс. Это делает Node.js удобным фундаментом для инструментов сборки и дев‑серверов, которые должны быть быстрыми и отзывчивыми.

## Почему Node.js используется в этом проекте
В этом проекте Node.js нужен не для «серверной логики», а для инструментов разработки:
- запуск Vite dev‑сервера (`dev`);
- запуск сборки (`build`) и предпросмотра (`preview`);
- работа Sass‑компилятора;
- управление зависимостями через npm или yarn.

Другими словами, сайт остаётся статическим, но инструменты, которые его собирают и запускают, работают на Node.js.

## Модель событий и неблокирующий ввод‑вывод (кратко)
Node.js использует событийный цикл (event loop). Скрипт запускается, выполняет синхронные шаги, затем «входит» в цикл событий и обрабатывает асинхронные задачи. В отличие от систем с явным запуском цикла, Node.js автоматически остаётся в event loop, пока есть незавершённые операции. Такой подход позволяет строить быстрые инструменты для разработки и сборки.

## LTS и почему это важно
Node.js выпускается по расписанию: новые версии выходят регулярно, а чётные версии переходят в LTS (long‑term support) и получают длительную поддержку безопасности. Для учебных и рабочих проектов рекомендуется использовать именно LTS‑версии, чтобы инструменты сборки работали стабильно.

## Как проверить установленную версию
```bash
node -v
npm -v
```

Ожидается актуальная LTS‑версия Node.js. Если версия сильно устарела, могут возникать ошибки при установке зависимостей или запуске dev‑сервера.

## Как Node.js «запускает» проект
Когда вы выполняете команду:
```bash
npm run dev
```
или
```bash
yarn dev
```
Node.js читает `package.json`, находит скрипт `dev`, запускает Vite и держит процесс активным. Vite в свою очередь собирает модульный граф, компилирует Sass и отдает результат в браузер.

## Мини‑пример: что делает `npm run dev`
1. Node.js запускает менеджер пакетов.
2. Менеджер читает `package.json`.
3. Скрипт `dev` вызывает `vite`.
4. Vite запускает dev‑сервер.

Это не «магия», а последовательность команд, которые запускаются Node.js.

## Типовые ошибки и причины
- **Слишком старая версия Node.js** → зависимости не ставятся, Vite не запускается.
- **Node.js не в PATH** → `node: command not found`.
- **Разные версии Node.js в группе** → разные результаты сборки и предупреждений.

## Как понять, что всё работает
- `node -v` и `npm -v` выполняются без ошибок.
- `npm run dev` или `yarn dev` запускают Vite.
- В браузере открывается `http://localhost:5173` (или другой порт от Vite).
