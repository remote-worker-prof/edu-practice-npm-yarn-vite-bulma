# Node.js — гайд

## Что это
Node.js — среда выполнения JavaScript вне браузера. Она не делает сайт «серверным» сама по себе, а позволяет запускать инструменты сборки, тесты, линтеры и скрипты, которые нужны в современном фронтенде.

## Роль в этом проекте
Node.js нужен, чтобы:
- устанавливать зависимости (через npm или yarn);
- запускать Vite (`dev`, `build`, `preview`);
- компилировать Sass (через зависимость `sass`).

## Мини‑теория
JavaScript обычно исполняется в браузере. Node.js переносит это исполнение в терминал. Благодаря этому мы можем писать инструменты сборки на JS и запускать их одной командой. В учебном проекте Node.js — это «фундамент», без которого не работают npm/yarn и Vite.

## Проверка установки
```bash
node -v
npm -v
```

Ожидается LTS‑версия Node.js (например, `v20.x` и выше).

## Типовые сценарии
- «Не запускается dev‑сервер» → проверить версию Node.js.
- «Команда не найдена» → Node.js не в `PATH`.

## Мини‑пример
Если в проекте есть `package.json`, то команда
```bash
npm run dev
```
запускает Vite. Это работает только если установлен Node.js и npm.

## Типовые ошибки
- Слишком старая версия Node.js.
- Разные версии Node.js у разных участников → разное поведение.
- Установка Node.js без добавления в `PATH`.

## Как проверить, что всё работает
- `node -v` и `npm -v` выполняются без ошибок.
- `npm run dev` запускает Vite.

## Короткий вывод
Node.js — это «двигатель» для инструментов сборки. Сам сайт остаётся статическим, но без Node.js его не собрать и не запустить dev‑сервер.
